## Overview

A comprehensive breakdown of the modern AI developer stack, showing how to combine multiple AI tools into a cohesive workflow. The session covered when to use each tool (ChatGPT, Perplexity, Lovable, Replit, Codex, Claude Code) and how Cursor serves as the central hub for AI-assisted development.

## Format

- **Type**: In-person meetup
- **Venue**: Mozn Office, Ankara
- **Duration**: Evening session
- **Speaker**: 1
- **Structure**: Presentation + tool comparisons + workflow demos + Q&A
- **Audience**: Developers interested in optimizing their AI toolchain

## Highlights

- Comprehensive mapping of the AI tools landscape for developers
- Head-to-head comparisons of different AI coding tools
- Practical workflow demonstrations showing tool combinations
- Real examples of building SaaS products with AI assistance
- Making "impossible" projects feel tractable with the right tools

## Topics Covered

### AI Tools Landscape
- **ChatGPT**: General-purpose AI, brainstorming, documentation
- **Perplexity**: Research, fact-checking, finding libraries/solutions
- **Lovable**: Rapid UI prototyping and design
- **Replit**: Quick deployment, collaborative coding
- **Codex/Claude Code**: Terminal-based AI coding agents
- **Cursor**: Central IDE integrating AI across the workflow

### Workflow Patterns
- Research phase: Perplexity for discovery, ChatGPT for planning
- Design phase: Lovable for UI mockups
- Development phase: Cursor for implementation
- Deployment phase: Replit for quick hosting
- Iteration: Combining tools based on task needs

### Best Practices
- Choosing the right tool for each task
- Avoiding tool switching overhead
- Building personal workflows that stick
- Balancing speed vs. understanding

## Key Takeaways

- No single AI tool is best for everything - build a stack
- Cursor excels as the development hub but benefits from other tools
- Research (Perplexity) + Planning (ChatGPT) + Building (Cursor) = powerful combo
- Time invested in learning tools pays off exponentially
- "Impossible" projects become tractable with proper AI tool orchestration
