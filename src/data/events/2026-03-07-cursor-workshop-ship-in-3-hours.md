## Overview

A 3-hour interactive workshop where participants built and deployed real software using Cursor. The event featured a guest presentation on the latest Cursor features, practical workflow tips, and a hands-on building session where attendees shipped working prototypes.

## Format

- **Type**: In-person workshop (with online attendance)
- **Venue**: Leqaa Hall, Ankara
- **Co-organizer**: DAL | Türkçe
- **Duration**: 11:00 PM – 2:00 AM (3 hours)
- **Speakers**: 3 ([Emre Özcan](https://x.com/buildermare), [Nick Miller](https://x.com/nickwm), [Abdullah Moasibah](https://www.linkedin.com/in/abdullah-mosaibah-67814312b/))
- **Structure**: Introduction → Guest speaker (Nick Miller) → Tricks & Tips → Build Time (1 hour)
- **Audience**: 70 in-person + 10 online (out of 110 accepted, 320+ registrations)

## Highlights

- Nick Miller presented the latest Cursor features (Skills, Hooks, Marketplace, Automation) with clear examples and actionable tips
- Practical session on faster workflows, better prompting habits, and moving from idea to product quickly
- Attendees built and deployed real prototypes during the hands-on segment

## Topics Covered

### Guest Speaker: Nick Miller (Engineer at Cursor)
- Latest Cursor features: Skills, Hooks, Marketplace, and Automation
- Practical Q&A on real-world usage and AI in production

### Tricks & Tips: Do's and Don'ts
- Vibe Code vs Vibe Engineer (Research → Plan → Build → Review)
- Review everything — treat AI as a junior dev
- Use the right model for the job (planning vs execution vs scaffolding)
- Don't start from scratch — use Lovable, v0, or templates first
- Use @docs for up-to-date library documentation
- Use Cursor Rules (User Rules + Project Rules) for persistent context
- Use MCPs, Skills & Extensions (Perplexity, Context7, TaskMaster, Mobile)
- Keep conversations focused — one task per chat
- Plan mode first, Agent mode second
- Write specific, context-rich prompts with @file, @folder, @codebase
- Checkpoint often — git commits + Cursor checkpoints
- Parallelize with Background Agents

### AI Tool Pipeline
- Research (ChatGPT/Perplexity) → Prototype (Replit/Lovable/v0) → Build (Cursor) → Polish (variant.com/v0) → Ship

### Hands-on Building
- Attendees built apps including: a pet-feeding tracker with notifications and shared logging, a social media image generation tool, and a Ankara bicycle community app
- MCP integrations used: PostHog MCP for user journey analysis, database access MCP for product insights, HexStrike MCP for vulnerability analysis
